SRS > Use Case Suite > Use Case Format

Process impact: This reference page documents the format of use cases and gives tips on writing use cases. You can copy and paste the sample use case into your Use Cases document. This file itself should not be edited to hold specific use cases.

Aspects common to all use cases

Direct Actors:
User: end-user in any role
System: The system being built
When actors are not listed, assume User is doing it.
Items beginning with "see" indicate that System has presented a new screen.
Stakeholders: The user who is entering the data, and those who will read it
Prereq: Project is set up

UC-00: Configure the site

Summary: The administrator navigates to the site configuration page and uses it to change the behavior of the web application.
Priority: Expected
Use Frequency: Rarely
Direct Actors: System, Administrator
Stakeholders: executives/interlectual property owner, User
Prereq:
Project is set up
Main Success Scenario:
  1. visit admin configuration page
  2. Update page data
  3. confim changes with the admin configuration page
Alternative Scenario Extensions:
  • N/A
Notes and Questions
  • How will administrators know the information has been updated? They could compare them with the bus agencies offical travel timetable and match them and make the nessercary changes.

UC-01: List travel agencies

Summary: The web app displays a clear list of travel agencies (bus/train/tram)
Priority: Essential
Use Frequency: Always
Direct Actors: System, User, administrator
Stakeholders: Interlectual property owner/executives, User
Prereq:
Project is set up
Main Success Scenario:
  1. Display travel agencies
  2. select any travel agency from the list
  3. display selected travel agency on top of list
Alternative Scenario Extensions:
  • If CONDITION, then ALTERNATIVE STEPS.
    • None.

UC-02: Check Stop

Summary: A text field for entering a stop name for a transport agency
Priority: Essential
Use Frequency: Always
Direct Actors: User, Administrator
Stakeholders: Interlectual property owner/executives, User
Prereq:
Project is set up
uc-01 has to be successfully completed
Main Success Scenario:
  1. display textfield for stop name entry
  2. enter a stop within text field
  3. submit the stop name
  4. validate stop name
Alternative Scenario Extensions:
  • If CONDITION, then ALTERNATIVE STEPS.
    • validation fails then restart process.
Notes and Questions
  • we could list the stops with images
